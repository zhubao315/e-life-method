Theory

四大理论基石

从 5,000+ 篇笔记的实战中提炼，经 20+ 轮审计迭代验证。

知识复利理论

知识的价值不在于拥有多少，而在于连接多少。每篇笔记都是本金，每次连接都是利息。连接密度决定复利价值——100 篇互相链接的笔记，价值远超 100 篇孤立笔记。

四维身份模型

人不是单一角色。知识应该按"你是谁"来组织——架构师关注如何构建，投资人关注值不值得，学习者关注为什么，生活艺术家关注怎么活。

知识转化阶梯

从原始输入到原创洞察，经历六层提炼：原始输入 → 概念 → 模型 → 框架 → 原则 → 洞察。每一层提炼都增加可迁移性和复利价值。

人机共生理论

AI 擅长结构化处理，人类擅长意义理解。最优解不是人或机器独立完成，而是人机协作：人类制定规则，AI 执行维护，AI 报告异常，人类决策处理。

Method

四大系统方法

可迁移的方法论，不限于特定工具。

空间架构法

知识库是分层空间，每层有明确功能定位。

  • 规则层（META）— 定义系统如何运转
  • 思维层（BRAIN）— 提炼后的知识资产
  • 产出层（BUILD）— 正在推进的项目
  • 身份层 — 按四维身份组织的领域知识
  • 输入层（SOURCES）— 原始信息和素材
  • 时间层（CALENDAR）— 按时间组织的记录

双轴标签法

用两个正交的标签轴为每篇笔记定位。

  • 维度轴（身份）— 架构/投资/学习/生活/管理
  • 类型轴（形态）— 概念/模型/框架/原则/洞察
  • 来源轴（可选）— 阅读/项目/媒体/经历
  • 项目轴（可选）— uRead/uSkill/uWise

MOC 导航法

Map of Content 是知识库的导航地图。

  • 顶层 MOC — 领域入口索引（∑ 主题.md）
  • 目录 README — 每个目录的索引
  • 主题 MOC — 特定主题的聚合索引
  • 用链接而非文件夹实现多归属

项目驱动法

用项目驱动知识的转化和应用。

  • uRead — 读书笔记产品化（输入→加工）
  • uSkill — 技能卡片库（实践→输出）
  • uWise — 知识百科（跨领域融合）
  • 每日推进，每周复盘，持续迭代

Practice

日常实践指南

从理论到行动，具体可执行的操作步骤。

Morning

晨间捕获

创建当日日记，记录晨间仪式（冥想、阅读、运动），捕获前夜灵感。每天的第一笔知识投资。

Daytime

日间积累

读到好内容 → 创建读书笔记。产生想法 → 创建洞察笔记。学到技能 → 创建技能卡片。每篇至少建立 2 个链接。

Evening

晚间复盘

更新日记完成情况，提炼当日关键洞察，检查待办事项。让每一天都有迹可循。

Weekly

周度整理

回顾本周笔记，建立新连接，检查三大项目进度，更新 MOC 索引。每周日 20:00 固定执行。

Monthly

月度维护

运行断链巡检、标签审计、骨架文件清理。更新知识库健康度评分。预防优于修复。
